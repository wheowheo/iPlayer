# 프레임 드롭 분석 및 해결

## 테스트 환경
- Apple M1, 16GB RAM
- macOS 26.4
- 디스플레이: 2560×1600 60Hz

## 드롭 원인 분석

### 4K 60fps에서 발생하는 문제

```
프레임 간격: 16.67ms (60fps)
평균 디코딩: 14.0ms → 여유 2.7ms
I-프레임 디코딩: 최대 153ms → 약 9프레임 분량
```

### 원인 1: 단일 스레드 병목 (수정됨)

**수정 전 구조:**
```
[Read Thread] ← demux + video decode + audio decode 전부 1스레드
   ├─ readPacket()          ~0.1ms
   ├─ videoDecoder.decode() ~14ms (I-프레임 시 최대 153ms)
   └─ audioDecoder.decode() ~0.5ms
```

I-프레임 디코딩(153ms) 동안:
- 오디오 패킷을 읽지 못함 → 오디오 끊김
- 새 비디오 프레임이 큐에 들어오지 않음 → 큐 고갈 → 프레임 드롭

**수정 후 구조:**
```
[Demux Thread]     ← 패킷 읽기 + 오디오 디코딩 (경량)
   ├─ readPacket()          ~0.1ms
   ├─ 비디오 패킷 → 패킷 큐
   └─ audioDecoder.decode() ~0.5ms

[Video Decode Thread] ← 비디오 디코딩 전용 (중량)
   └─ videoDecoder.decode() ~14ms (I-프레임 시 153ms)

[Render Thread / CVDisplayLink]
   └─ 프레임 큐에서 적정 프레임 선택 및 표시
```

I-프레임 디코딩 중에도 오디오는 계속 처리되고, 이미 디코딩된 프레임 큐(최대 181 프레임 = 3초)가 버퍼 역할.

### 원인 2: 큐 고갈 (수정됨)

**수정 전**: 큐 배압 임계값 120에서 읽기 중단 → I-프레임 스파이크 시 120프레임이 2초 만에 고갈
**수정 후**: 프레임 큐와 패킷 큐 분리, 패킷 큐 120개로 선행 읽기 → 큐 고갈 0회

### 원인 3: 프리버퍼링 부재 (수정됨)

**수정 전**: `Thread.sleep(0.15)` 고정 대기 후 렌더 시작 → 큐에 프레임 부족한 상태로 시작
**수정 후**: 목표 프레임 수(60fps: 30프레임=0.5초) 달성 시 렌더 시작 → 안정적 시작

### 원인 4: Main Thread Dispatch 지연

`DispatchQueue.main.async`로 프레임을 전달하면, 메인 런루프가 바쁠 때 최대 55ms 지연 발생.
CVDisplayLink 모드에서는 vsync 타이밍이 정확하므로 지연이 최소화됨.

## 적용된 해결책

### 1. Demux/Decode 스레드 분리 (핵심)
- `iPlayer.Demux`: 패킷 읽기 + 오디오 디코딩
- `iPlayer.VideoDecode`: 비디오 디코딩 전용
- 비디오 패킷 큐(용량 120)로 두 스레드 간 비동기 통신
- 세마포어 기반 패킷 전달 (대기/깨움)

### 2. 적응적 프리버퍼링
- 재생 시작 시 목표 프레임 수까지 큐를 채운 후 렌더 시작
- 60fps: 30프레임(0.5초), 30fps: 15프레임(0.5초)
- 최대 1초 타임아웃

### 3. VideoToolbox extra_hw_frames
- `extra_hw_frames = 8`로 HW surface 여유분 확보
- I-프레임 버스트 시 surface 경합 감소

### 4. FrameRingBuffer 개선
- `append()` → `Bool` 반환 (큐 포화 감지)
- `isFull` 프로퍼티 추가
- 디코딩 스레드에서 큐 포화 시 대기 후 재시도 (silent drop 방지)

### 5. CVDisplayLink EOF 크래시 수정
- `displayLinkRunning` 플래그를 콜백 진입부에서 체크
- `stopDisplayLink()`에서 release를 비동기로 지연 (콜백 스레드 종료 대기)

## 잔여 이슈

### Thread 모드 dispatch latency
Thread 모드에서 `DispatchQueue.main.async` max latency가 55ms에 달함.
CVDisplayLink 모드에서는 13ms 이하. **CVDisplayLink 사용 권장**.

### Decode Slow 경고 빈도
4K 60fps에서 디코딩 avg 14ms / limit 16.7ms로 여유가 2.7ms뿐.
이것은 HW 디코딩의 한계이며, 경고 임계값(80%)이 너무 민감할 수 있음.
실제 드롭과 무관한 경고이므로 임계값을 95%로 조정하는 것을 고려.

### 향후 개선
- Metal 렌더링: `CALayer.contents` 대신 `CAMetalLayer` + `CVMetalTextureCache`로 main thread dispatch 제거
- 다중 디코더: 소프트웨어 디코딩 시 `thread_count` 활용
